# ADR-002: 推荐策略选择

## 状态

已接受

## 上下文

在ADR-001中，我们决定采用陪伴型助手模式，其中智能推荐助手是核心模块。现在需要决定推荐算法策略以满足不同用户群体的需求。

基于第一阶段需求调研，用户需求特征如下：
- **考研学生**：需要资源主动推荐（H-KY05），紧急程度高，影响范围中等
- **留学生**：需要英文文献导航（H-IL01）和跨语言学习支持（H-IL02），紧急程度高
- **教师**：需要新书自动追踪和推送（H-T01），紧急程度高
- **新生**：需要学习路径规划（H-F03），紧急程度低

推荐策略需要考虑以下维度：
1. **公平性**：避免推荐算法偏向热门资源，忽视小众需求
2. **多样性**：推荐内容需涵盖不同类型、不同领域的资源
3. **个性化**：根据用户身份、历史行为提供个性化推荐
4. **情境感知**：根据用户当前情境（如考试周、座位紧张度）调整推荐

## 决策

我们选择**情境感知混合推荐策略**，结合协同过滤、内容匹配和情境感知三种方法。

### 选项分析

#### 选项A：协同过滤推荐
**定义**：基于用户历史行为数据（借阅记录、座位使用记录、搜索记录）和相似用户行为进行推荐

**优点**：
- 算法成熟，易于实现
- 能发现用户未知兴趣
- 用户群体相似性高（同校学生）时效果显著

**缺点**：
- **公平性问题**：容易偏向热门资源，冷门资源推荐不足
- **冷启动问题**：新用户无历史行为数据，无法推荐
- **多样性不足**：推荐结果可能集中于某一类资源
- 数据依赖性强，需大量历史数据

**适用场景**：
- 用户行为数据丰富
- 用户群体相似性高
- 不考虑小众需求场景

#### 选项B：内容匹配推荐
**定义**：基于资源特征（书籍主题、学科分类、语言类型）和用户兴趣特征进行匹配推荐

**优点**：
- 无冷启动问题，可根据用户注册信息推荐
- **公平性较好**：能推荐冷门资源（只要符合用户兴趣）
- 推荐结果可解释性强
- 不依赖其他用户行为数据

**缺点**：
- 推荐结果受限于用户已知兴趣，无法发现新兴趣
- 需要高质量资源标注数据
- 无法捕捉用户潜在兴趣
- **多样性不足**：推荐结果可能过于狭窄

**适用场景**：
- 新用户场景（无历史行为数据）
- 资源标注质量高
- 用户兴趣明确且稳定

#### 选项C：情境感知推荐
**定义**：根据用户当前情境（如考试周、座位紧张度、天气、时间）动态调整推荐策略

**优点**：
- 推荐结果更贴近用户当前需求
- 能捕捉用户动态需求变化
- 推荐时机精准，用户接受度高

**缺点**：
- 需要实时情境数据，技术复杂度高
- 情境数据准确性依赖外部数据源（如天气API）
- 情境维度设计复杂
- 推荐结果不稳定，难以长期优化

**适用场景**：
- 用户需求随情境变化显著
- 有实时情境数据源
- 技术团队能力强

#### 选项D：混合推荐策略
**定义**：结合协同过滤、内容匹配、情境感知多种方法，根据不同场景动态切换

**优点**：
- **克服单一方法缺点**：
  - 协同过滤解决冷启动问题（结合内容匹配）
  - 内容匹配发现新兴趣（结合协同过滤）
  - 情境感知动态调整推荐时机
- **公平性和多样性可控**：通过调整混合权重平衡热门与冷门
- **适应不同用户群体**：根据用户特征选择不同策略组合

**缺点**：
- 技术架构复杂，需要集成多种算法
- 推荐结果可解释性降低
- 参数调优难度大
- 维护成本高

**适用场景**：
- 用户需求多样化
- 技术团队规模大
- 追求推荐质量最优

### 决策理由

我们选择**情境感知混合推荐策略**，具体设计如下：

#### 混合策略框架

```
┌─────────────────────────────────────────┐
│          用户请求                        │
│  用户ID + 情境信息 + 推荐类型            │
└─────────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│          情境分析模块                    │
│  当前情境：考试周、座位紧张度、时间等    │
└─────────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│          策略选择模块                    │
│  根据情境选择推荐策略组合                │
└─────────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│          多策略并行计算                  │
│  协同过滤 | 内容匹配 | 情境感知          │
└─────────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│          结果融合与排序                  │
│  加权融合 + 公平性修正 + 多样性保证      │
└─────────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│          推荐结果输出                    │
│  推荐列表 + 推荐理由 + 相关信息          │
└─────────────────────────────────────────┘
```

#### 策略组合设计

##### 策略组合1：新用户推荐（冷启动场景）
**适用用户**：新生、新注册用户（无历史行为数据）

**策略组合**：
- **内容匹配（权重70%）**：基于用户注册信息（专业、年级、兴趣标签）推荐
- **热门推荐（权重20%）**：推荐图书馆热门资源（确保推荐质量）
- **情境感知（权重10%）**：根据入学时间推荐新生引导资源

**公平性修正**：
- 热门推荐中限制同一类别资源数量（≤30%）
- 强制推荐不同学科资源（至少涵盖3个学科）

**多样性保证**：
- 推荐结果涵盖：书籍、座位、活动、资源平台
- 每种类型至少推荐1项

##### 策略组合2：考研学生推荐（长期备考场景）
**适用用户**：考研学生（标签：考研 + 备考科目）

**策略组合**：
- **内容匹配（权重50%）**：基于考研科目推荐相关书籍和资料
- **协同过滤（权重30%）**：基于其他考研学生行为推荐（相似备考科目）
- **情境感知（权重20%）**：根据备考阶段（如冲刺阶段）调整推荐强度

**公平性修正**：
- 协同过滤中限制热门考研资料推荐数量（避免过度推荐）
- 强制推荐小众考研资料（如专业课冷门书籍）

**多样性保证**：
- 推荐结果涵盖：考研书籍、网课资源、备考经验、心理支持内容
- 不同科目资源比例平衡（避免单一科目）

##### 策略组合3：留学生推荐（跨语言场景）
**适用用户**：留学生（标签：国籍 + 中文水平 + 专业）

**策略组合**：
- **内容匹配（权重60%）**：基于专业和中文水平推荐适合文献
- **语言过滤（权重20%）**：优先推荐英文文献，辅助推荐中文文献+翻译支持
- **情境感知（权重20%）**：根据课程进度推荐课程相关文献

**公平性修正**：
- 强制推荐不同语言资源（英文40%、中文40%、双语20%）
- 避免"过度推荐英文文献，忽视中文文献学习需求"

**多样性保证**：
- 推荐结果涵盖：英文文献、中文文献（带翻译）、跨文化学习资料、语言学习资源

##### 策略组合4：教师推荐（学术追踪场景）
**适用用户**：教师（标签：研究方向 + 教授课程）

**策略组合**：
- **内容匹配（权重40%）**：基于研究方向推荐最新学术文献
- **新书追踪（权重40%）**：推送图书馆新购入书籍（研究方向相关）
- **协同过滤（权重20%）**：基于相似研究方向教师推荐

**公平性修正**：
- 新书追踪中确保小众研究方向书籍推送（避免仅推送热门研究方向）
- 强制推荐跨学科资源（促进跨学科研究）

**多样性保证**：
- 推荐结果涵盖：学术文献、教学资源、跨学科资料、学术活动

##### 策略组合5：焦虑学生推荐（情感支持场景）
**适用用户**：焦虑学生（标签：情绪状态 + 学习压力）

**策略组合**：
- **情境感知（权重50%）**：根据当前情绪状态调整推荐（如压力大时推荐心理支持内容）
- **内容匹配（权重30%）**：基于学习需求推荐安静座位、轻松学习资料
- **协同过滤（权重20%）**：基于相似状态学生推荐（如成功应对压力的学生）

**公平性修正**：
- 强制推荐心理支持内容（避免仅推荐学习资料）
- 避免"过度推荐热门座位，忽视安静角落"

**多样性保证**：
- 推荐结果涵盖：安静座位、轻松学习资料、心理支持内容、同伴支持网络

#### 公平性保障机制

##### 机制1：热门资源限制
**问题**：协同过滤容易过度推荐热门资源

**解决方案**：
- 设置热门资源推荐上限（每种类型≤30%）
- 强制推荐冷门资源（至少20%）

**算法实现**：
```python
# 热门资源限制算法伪代码
def ensure_fairness(recommendations):
    # 统计热门资源比例
    popular_ratio = count_popular(recommendations) / len(recommendations)
    
    # 如果热门资源超过30%，替换为冷门资源
    if popular_ratio > 0.3:
        recommendations = replace_with_niche(recommendations, 0.3)
    
    # 确保冷门资源至少20%
    niche_ratio = count_niche(recommendations) / len(recommendations)
    if niche_ratio < 0.2:
        recommendations = add_niche(recommendations, 0.2)
    
    return recommendations
```

##### 机制2：小众需求保障
**问题**：小众需求（如留学生英文文献、焦虑学生心理支持）容易被忽视

**解决方案**：
- 针对特定群体强制推荐小众需求资源
- 设置小众需求推荐权重下限

**算法实现**：
```python
# 小众需求保障算法伪代码
def ensure_niche_needs(user_type, recommendations):
    if user_type == "international_student":
        # 确保英文文献推荐
        english_ratio = count_english_books(recommendations) / len(recommendations)
        if english_ratio < 0.4:
            recommendations = add_english_books(recommendations, 0.4)
    
    elif user_type == "anxious_student":
        # 确保心理支持推荐
        mental_ratio = count_mental_support(recommendations) / len(recommendations)
        if mental_ratio < 0.2:
            recommendations = add_mental_support(recommendations, 0.2)
    
    return recommendations
```

#### 多样性保障机制

##### 机制1：资源类型多样性
**问题**：推荐结果可能集中于单一资源类型（如仅推荐书籍）

**解决方案**：
- 强制推荐多种资源类型（书籍、座位、活动、资源平台）
- 设置每种资源类型推荐数量下限

**算法实现**：
```python
# 资源类型多样性算法伪代码
def ensure_diversity(recommendations):
    types = ["book", "seat", "activity", "resource"]
    
    for type in types:
        ratio = count_type(recommendations, type) / len(recommendations)
        if ratio < 0.1:  # 每种类型至少10%
            recommendations = add_type(recommendations, type, 0.1)
    
    return recommendations
```

##### 机制2：学科领域多样性
**问题**：推荐结果可能集中于单一学科领域

**解决方案**：
- 强制推荐跨学科资源（至少涵盖3个学科）
- 设置跨学科推荐权重

**算法实现**：
```python
# 学科多样性算法伪代码
def ensure_subject_diversity(recommendations):
    subjects = extract_subjects(recommendations)
    
    # 确保至少3个学科
    if len(subjects) < 3:
        recommendations = add_cross_subject(recommendations, 3)
    
    # 确保跨学科资源至少20%
    cross_ratio = count_cross_subject(recommendations) / len(recommendations)
    if cross_ratio < 0.2:
        recommendations = add_cross_subject(recommendations, 0.2)
    
    return recommendations
```

### 决策依据

1. **需求匹配度**：
   - 不同用户群体需求差异显著，单一策略无法满足
   - 混合策略能适应不同用户群体特征

2. **公平性考虑**：
   - 协同过滤偏向热门资源，需通过混合策略修正
   - 情境感知混合策略能保障小众需求

3. **多样性考虑**：
   - 单一策略推荐结果多样性不足
   - 混合策略能强制保障多样性

4. **技术可行性**：
   - 现有推荐算法库（如Surprise、LightFM）支持混合策略
   - 技术团队有能力实现和调优

### 决策后果

#### 正面后果
1. 推荐质量提升：混合策略克服单一策略缺点
2. 公平性保障：避免热门资源过度推荐，小众需求得到满足
3. 多样性保证：推荐结果涵盖多种类型、多个学科
4. 用户满意度提升：个性化推荐贴近用户真实需求

#### 负面后果
1. 技术架构复杂：需集成多种推荐算法
2. 参数调优难度大：需不断优化权重组合
3. 维护成本高：需持续监控推荐质量
4. 推荐结果可解释性降低：用户难以理解推荐理由

#### 风险缓解措施
1. 分阶段实施：先实现单一策略，逐步叠加混合策略
2. 推荐理由展示：为用户提供推荐理由解释
3. 用户反馈机制：收集用户反馈优化推荐权重
4. 推荐质量监控：建立推荐质量评估体系

## 技术实现概览

### 推荐算法模块

```python
# 推荐系统核心模块设计
class HybridRecommender:
    def __init__(self):
        self.collaborative_filter = CollaborativeFilter()
        self.content_matcher = ContentMatcher()
        self.context_sensor = ContextSensor()
    
    def recommend(self, user_id, context):
        # 情境分析
        situation = self.context_sensor.analyze(context)
        
        # 策略选择
        strategy_weights = self.select_strategy(user_id, situation)
        
        # 多策略并行计算
        cf_results = self.collaborative_filter.recommend(user_id)
        content_results = self.content_matcher.recommend(user_id)
        context_results = self.context_sensor.recommend(user_id, situation)
        
        # 结果融合
        hybrid_results = self.merge_results(
            cf_results, content_results, context_results, 
            strategy_weights
        )
        
        # 公平性修正
        fair_results = self.ensure_fairness(hybrid_results, user_id)
        
        # 多样性保证
        diverse_results = self.ensure_diversity(fair_results)
        
        return diverse_results
```

## 验证标准

### 推荐质量指标
- **命中率（Hit Rate）**：用户实际使用推荐资源的比例≥30%
- **覆盖率（Coverage）**：推荐资源覆盖图书馆资源比例≥60%
- **多样性指数（Diversity Index）**：推荐结果多样性评分≥0.7

### 公平性指标
- **小众需求满足率**：小众需求推荐比例≥20%
- **热门资源限制率**：热门资源推荐比例≤30%
- **跨学科推荐比例**：跨学科资源推荐比例≥20%

### 用户满意度指标
- **推荐满意度评分**：用户对推荐结果评分≥4.0（5分制）
- **推荐使用率**：用户实际使用推荐资源比例≥30%
