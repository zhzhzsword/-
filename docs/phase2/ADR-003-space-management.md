# ADR-003: 空间管理策略选择

## 状态

已接受

## 上下文

在第一阶段需求调研中，我们发现用户对座位的需求远超"查找空位"这一显性需求：
- **考研学生**：需要固定座位的归属感（H-KY01）、座位临时保留功能（H-KY02），优先级得分11和12，排名前列
- **焦虑学生**：需要安静角落优先分配（H-A03），优先级得分7
- **教师**：需要安静独立工作空间（H-T04），优先级得分10
- **新生**：需要使用引导（H-F01），包含座位使用教学

现有座位管理系统存在的问题：
1. **分配僵化**：仅显示空位数量，无座位特征识别
2. **公平性问题**：考研学生长期占用座位，其他学生不公平
3. **利用率低**：角落座位利用率高，但无优先分配机制
4. **临时离开问题**：考研学生用餐时座位被占，导致崩溃

需要决定空间管理策略以满足不同用户群体的隐性需求，同时保障公平性。

## 决策

我们选择**动态调度 + 固定分配混合模式**，为不同用户群体提供差异化的座位管理策略。

### 选项分析

#### 选项A：固定分配模式
**定义**：为特定用户群体（如考研学生）分配固定座位，长期使用

**优点**：
- 满足考研学生对"归属感"和"控制感"的需求
- 考研学生满意度显著提升
- 减少抢座竞争和焦虑

**缺点**：
- **公平性问题**：其他学生认为考研学生特权，引发不满
- **利用率问题**：考研学生不使用时座位空置，利用率低
- **管理复杂**：需要审核考研学生资格、分配座位、监督使用
- **资源浪费**：固定座位无法动态调整，资源浪费

**适用场景**：
- 考研学生需求明确且强烈
- 座位资源充足
- 管理团队有能力监督

#### 选项B：动态调度模式
**定义**：所有座位动态分配，根据实时需求和使用情况调度

**优点**：
- **公平性好**：所有学生平等竞争座位
- **利用率高**：座位实时调度，减少空置
- **管理简单**：无需固定分配审核，系统自动调度
- **资源优化**：根据实时需求优化座位分配

**缺点**：
- 无法满足考研学生对"归属感"的需求
- 考研学生每天抢座，焦虑感持续
- 临时离开时座位被占，崩溃问题未解决

**适用场景**：
- 用户需求无明显差异
- 座位资源紧张
- 追求公平和利用率

#### 选项C：混合模式（固定分配 + 动态调度）
**定义**：部分座位固定分配给特定用户群体，其余座位动态调度

**优点**：
- **兼顾不同需求**：
  - 固定分配满足考研学生归属感需求
  - 动态调度保障其他学生公平性
- **资源优化**：固定座位不使用时可临时释放
- **灵活性**：可根据用户需求动态调整固定座位比例

**缺点**：
- 管理复杂度增加：需同时管理固定和动态座位
- 技术架构复杂：需开发固定分配和动态调度两套机制
- 公平性平衡困难：需平衡固定与动态座位比例

**适用场景**：
- 用户需求差异显著
- 座位资源中等规模
- 技术团队能力强

### 决策理由

我们选择**动态调度 + 固定分配混合模式**，具体设计如下：

#### 混合策略框架

```
图书馆座位资源分层管理：
┌─────────────────────────────────────────┐
│          固定座位区域（20%）             │
│  专为考研学生、教师等长期需求用户       │
│  座位编号：A01-A40                      │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│          优先座位区域（30%）             │
│  为焦虑学生、留学生等特殊需求用户       │
│  座位编号：B01-B60                      │
│  特征：靠窗、靠墙、安静角落             │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│          动态座位区域（50%）             │
│  所有用户公平竞争                       │
│  座位编号：C01-C100                     │
│  实时调度，无固定分配                   │
└─────────────────────────────────────────┘
```

#### 分层策略设计

##### 第一层：固定座位区域（20%）
**目标用户**：考研学生（备考≥6个月）、教师

**管理策略**：
1. **申请审核机制**
   - 考研学生提交申请：备考计划、备考时长、备考科目
   - 教师提交申请：研究方向、使用频率
   - 系统审核：自动审核（备考时长≥6个月）+管理员人工审核

2. **固定座位分配**
   - 系统根据备考科目匹配座位（如数学备考学生分配在靠近数学书架区域）
   - 教师分配在教师阅览室独立空间

3. **使用监督机制**
   - 固定座位使用率监控：每周使用率低于60%则收回
   - 使用记录：记录学生使用时长、使用频率
   - 违规处理：长期不使用座位则收回分配权

4. **临时释放机制**
   - 考研学生临时离开（如用餐）时，可申请临时释放座位（≤30分钟）
   - 临时释放期间座位进入动态调度池，其他学生可临时使用
   - 返回后系统自动收回座位

##### 第二层：优先座位区域（30%）
**目标用户**：焦虑学生、留学生、新生

**管理策略**：
1. **座位特征识别**
   - 系统识别座位特征：靠窗、靠墙、安静角落、人流少区域
   - 标注座位属性：安静度、私密性、视野、便利性

2. **用户特征匹配**
   - 焦虑学生：优先分配安静角落、私密性强的座位
   - 留学生：优先分配靠近外文阅览室、人流少区域
   - 新生：优先分配靠近服务台、视野开阔的座位（便于观察学习）

3. **优先分配机制**
   - 用户进入图书馆时，系统自动识别用户类型
   - 根据用户类型推荐优先座位区域
   - 优先座位区域无空位时，推荐动态座位区域

##### 第三层：动态座位区域（50%）
**目标用户**：所有用户（公平竞争）

**管理策略**：
1. **实时调度算法**
   - 基于用户需求（如学习时长、座位特征偏好）推荐座位
   - 基于座位实时状态（空置时间、使用频率）优化分配
   - 基于情境（如考试周座位紧张）调整分配策略

2. **公平竞争机制**
   - 所有用户平等竞争动态座位
   - 无固定分配，无优先权
   - 实时排队机制：座位紧张时建立排队队列

3. **智能推荐机制**
   - 根据用户历史偏好推荐座位（如常选靠窗座位）
   - 根据用户当前情境推荐（如考试周推荐安静座位）
   - 根据图书馆实时状态推荐（如某区域拥挤则推荐其他区域）

#### 公平性保障机制

##### 机制1：固定座位比例限制
**问题**：固定座位比例过高，其他学生不公平

**解决方案**：
- 固定座位区域限制为20%（约40个座位）
- 根据用户需求动态调整固定座位比例（考试周时临时增加固定座位）

**算法实现**：
```python
# 固定座位比例动态调整算法伪代码
def adjust_fixed_ratio(situation):
    # 正常时期：固定座位20%
    if situation == "normal":
        fixed_ratio = 0.2
    
    # 考试周：临时增加固定座位至30%（满足考研学生冲刺需求）
    elif situation == "exam_week":
        fixed_ratio = 0.3
    
    # 假期：减少固定座位至10%（考研学生减少）
    elif situation == "vacation":
        fixed_ratio = 0.1
    
    return fixed_ratio
```

##### 机制2：固定座位使用监督
**问题**：固定座位长期不使用，资源浪费

**解决方案**：
- 监控固定座位使用率：每周使用率低于60%则收回
- 建立"使用积分"机制：使用率越高，积分越高，积分低于阈值则收回

**算法实现**：
```python
# 固定座位使用监督算法伪代码
def supervise_fixed_seat_usage(user_id, seat_id):
    # 计算每周使用率
    usage_ratio = calculate_weekly_usage(user_id, seat_id)
    
    # 使用率低于60%：警告提醒
    if usage_ratio < 0.6:
        send_warning(user_id, "固定座位使用率过低，请注意")
    
    # 使用率低于40%：收回座位
    if usage_ratio < 0.4:
        revoke_fixed_seat(user_id, seat_id)
        send_notification(user_id, "固定座位已收回")
    
    return usage_ratio
```

##### 机制3：优先座位透明化
**问题**：优先座位分配不透明，用户不理解

**解决方案**：
- 优先座位分配规则公开：用户可查看优先分配条件
- 优先座位分配结果公示：用户可查看谁获得优先座位及原因
- 用户申诉机制：对优先座位分配不满可申诉

#### 资源利用率优化机制

##### 机制1：座位临时释放
**问题**：固定座位用户临时离开时座位空置

**解决方案**：
- 固定座位用户可申请临时释放座位（≤30分钟）
- 临时释放期间座位进入动态调度池
- 其他用户可临时使用，但需明确"临时使用，可能被收回"
- 固定用户返回时系统自动收回座位

**算法实现**：
```python
# 座位临时释放算法伪代码
def temporary_release_seat(user_id, seat_id, duration):
    # 检查释放时长（≤30分钟）
    if duration > 30:
        return False, "临时释放时长不能超过30分钟"
    
    # 标记座位为临时释放状态
    mark_seat_temporary_released(seat_id, user_id, duration)
    
    # 座位进入动态调度池
    add_to_dynamic_pool(seat_id)
    
    # 设置自动收回计时器
    set_auto_reclaim_timer(seat_id, user_id, duration)
    
    return True, "座位已临时释放"
```

##### 机制2：动态座位智能调度
**问题**：动态座位分配效率低，用户不满意

**解决方案**：
- 基于用户历史偏好推荐座位
- 基于座位实时状态优化分配
- 基于情境调整分配策略

**算法实现**：
```python
# 动态座位智能调度算法伪代码
def dynamic_seat_scheduling(user_id, context):
    # 获取用户历史偏好
    user_preference = get_user_preference(user_id)
    
    # 获取座位实时状态
    seat_status = get_real_time_seat_status()
    
    # 分析当前情境
    situation = analyze_context(context)
    
    # 推荐座位
    if situation == "exam_week":
        # 考试周：优先推荐安静座位
        recommendations = recommend_quiet_seats(seat_status)
    elif situation == "vacation":
        # 假期：推荐任意空位
        recommendations = recommend_any_empty(seat_status)
    else:
        # 正常时期：根据用户偏好推荐
        recommendations = recommend_by_preference(seat_status, user_preference)
    
    return recommendations
```

### 决策依据

1. **需求匹配度**：
   - 考研学生对固定座位需求强烈（优先级得分11）
   - 焦虑学生对优先座位需求明确（优先级得分7）
   - 其他用户对公平性需求强烈

2. **公平性考虑**：
   - 固定分配引发公平性问题，需限制比例
   - 混合模式能平衡固定需求与公平需求

3. **资源利用率**：
   - 固定座位空置时资源浪费，需临时释放机制
   - 动态调度能提高资源利用率

4. **技术可行性**：
   - 固定分配和动态调度技术均可实现
   - 技术团队有能力开发混合管理机制

### 决策后果

#### 正面后果
1. 考研学生满意度提升：固定座位满足归属感需求
2. 焦虑学生心理安全感提升：优先分配安静角落
3. 公平性保障：动态座位区域公平竞争
4. 资源利用率提升：临时释放机制减少空置

#### 负面后果
1. 管理复杂度增加：需同时管理三层座位区域
2. 公平性争议风险：固定座位可能引发其他学生不满
3. 技术架构复杂：需开发固定分配、优先分配、动态调度三套机制
4. 监督成本增加：需监控固定座位使用率

#### 风险缓解措施
1. 公开分配规则：固定座位分配规则透明化
2. 用户教育：向用户解释混合管理模式必要性
3. 公平性审核机制：建立用户申诉机制
4. 动态调整比例：根据用户反馈调整固定座位比例

## 技术实现概览

### 空间管理模块

```python
# 空间管理核心模块设计
class SpaceManager:
    def __init__(self):
        self.fixed_allocator = FixedAllocator()
        self.priority_allocator = PriorityAllocator()
        self.dynamic_scheduler = DynamicScheduler()
    
    def allocate_seat(self, user_id, user_type, context):
        # 根据用户类型选择分配策略
        if user_type == "graduate_student":
            # 考研学生：固定座位分配
            result = self.fixed_allocator.allocate(user_id)
        
        elif user_type in ["anxious_student", "international_student", "freshman"]:
            # 特殊需求用户：优先座位分配
            result = self.priority_allocator.allocate(user_id, user_type)
        
        else:
            # 普通用户：动态座位调度
            result = self.dynamic_scheduler.schedule(user_id, context)
        
        return result
    
    def temporary_release(self, user_id, seat_id, duration):
        # 座位临时释放
        return self.fixed_allocator.temporary_release(user_id, seat_id, duration)
    
    def auto_reclaim(self, seat_id, user_id):
        # 自动收回座位
        return self.fixed_allocator.auto_reclaim(seat_id, user_id)
```

## 验证标准

### 公平性指标
- **固定座位比例**：固定座位比例≤20%（考试周≤30%）
- **用户满意度差异**：固定座位用户与动态座位用户满意度差异≤10%
- **申诉率**：用户对座位分配申诉率≤5%

### 资源利用率指标
- **座位利用率**：座位利用率≥80%
- **固定座位使用率**：固定座位每周使用率≥60%
- **临时释放成功率**：临时释放成功率≥90%

### 用户满意度指标
- **考研学生满意度**：考研学生对固定座位满意度≥80%
- **焦虑学生满意度**：焦虑学生对优先座位满意度≥70%
- **普通学生满意度**：普通学生对动态座位满意度≥60%
